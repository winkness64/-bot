## 1. 工单目标

这次不是测显式记忆指令，也不是测单条回复漂不漂亮。
本工单只验一件事：

> **I叔 在真实重启后，owner 私聊主线会不会重新老年痴呆。**

要验证的核心能力：

1. 重启后还能咬住当前任务
2. 插入干扰轮次后还能回到主线
3. 能复述最近结论 / 待办 / 决策
4. 必要时切 fallback 后，不会像失忆一样从零开始

---

## 2. 测试前提

执行人：秧秧
测试对象：owner 私聊链路
建议环境：真实 NoneBot 运行态，非纯 mock

前提要求：

- 已部署当前私聊会话状态改造版本
- 本轮代码已生效
- 进行一次**真实进程重启**后再开始验收
- 测试期间尽量不要夹杂无关大段聊天

说明：

- 这轮重点是**重启后真实行为**，不是单测通过截图
- 如果只在未重启热态里测，结论不算最终验收通过

---

## 3. 通过标准

满足下面 4 条，才算这轮主线过关：

1. 连续 5~10 轮后，仍能答出当前主线任务
2. 中间插入其他话题/工具结果后，仍能回到主线
3. 能答出“刚才定了什么 / 下一步做什么”
4. 若出现 fallback 或等价切换，主线信息不丢

失败判定：

- 把当前任务说错
- 把旧噪音当成主线
- 完全答不出最近决策/待办
- 一遇到切换或长对话就重新失忆

---

## 4. 建议测试步骤

### Phase A：重启后首轮确认

1. 重启服务
2. 等机器人恢复正常可回复
3. owner 私聊发送一条明确主线指令，例如：
   - `我们现在在验证私聊主线抗健忘，先记住这件事`
4. 再补一条任务目标，例如：
   - `目标是看你重启后会不会老年痴呆`

预期：

- 能正常接任务
- 不出现明显上下文断片
- 不把无关历史误认成当前任务

### Phase B：主线保持

连续发送 3~5 轮围绕同一任务的问题，例如：

- `我们现在主线是什么`
- `这轮要验什么`
- `你现在在帮我处理哪件事`
- `这件事的验收点有哪些`

预期：

- 回答稳定指向“重启后 owner 私聊主线抗健忘验收”
- 不能漂去显式记忆、旧 C3、其他历史修复线

### Phase C：插入干扰再回主线

中间插入 1~2 轮干扰内容，例如：

- 问一个无关小问题
- 插一句别的安排
- 让它简单总结一次别的事项

然后立即回拉主线：

- `别岔开，刚才主线是什么`
- `继续这个验收，下一步该看什么`

预期：

- 能从干扰轮次回来
- 不需要 owner 重新完整交代背景

### Phase D：结论与待办复述

追问：

- `刚才已经确认了什么`
- `下一步准备测什么`
- `现在待办还有哪些`
- `最近做的决定是什么`

预期：

- 能提炼出已确认结论、下一步动作、最近决策
- 内容应围绕当前工单，不应散成闲聊摘要

### Phase E：长一点的连续回合

把对话拉到 8~10 轮左右，再问：

- `说一下我们这轮主线`
- `如果现在交给别人测试，你会怎么概括`

预期：

- 仍能保住主线
- 不因轮次增长明显失忆

### Phase F：fallback 观察（可选但强烈建议）

如果现场能稳定观察到 fallback / 切换迹象，再补一枪：

- 在一次较长或较复杂回复后，继续追问主线
- 关注切换后是否还能知道：
  - 当前任务
  - 已确认结论
  - 下一步待办

预期：

- fallback 是换枪，不是换脑子
- 切过去后主线仍连续

## 7. 结果判定模板

### 通过

- 重启后主线可保持
- 干扰后可回拉
- 最近结论/待办/决策可复述
- 长轮次无明显失忆

可回报：

> 本轮重启验收通过，owner 私聊主线抗健忘能力基本站住。

### 不通过

典型不通过样例：

- 第 3~5 轮开始忘当前任务
- 一插干扰就回不到主线
- 能聊天但答不出“刚才决定了什么”
- 切换后上下文像被洗掉

可回报：

> 本轮重启验收未通过，当前仍存在主线漂移/会话失忆，需要继续查 session state / prompt 注入 / fallback 继承。
